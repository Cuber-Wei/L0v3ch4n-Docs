简单的格密码问题

约 986 字大约 3 分钟

CTFCryptoLattice

2025-03-12

利用格基约化的技术，我们可以求解一些加密脚本中含未知数的问题。

格密码一般是基于两个格上的难题建立的（SVP和CVP）

注

这边引入一个定理（Hermite 定理）： 这个定理揭示了格中最短向量的大概长度。 定理本体：对nnn维的格LLL，都包含一个非零向量v∈L\mathbf{v}\in Lv∈L，满足：

∣∣v∣∣≤ndet(L)1/n||\mathbf{v}||\le\sqrt{n}det(L)^{1/n} ∣∣v∣∣≤n​det(L)1/n

衍生出Hermite 常数γn\gamma_nγn​，有：

∣∣v∣∣2≤γndet(L)2/n||\mathbf{v}||^2\le\gamma_ndet(L)^{2/n} ∣∣v∣∣2≤γn​det(L)2/n

n2πe≤γn≤n2πe\frac{n}{2\pi e}\le\gamma_n\le\frac{n}{2\pi e} 2πen​≤γn​≤2πen​

格密码的解题思路一般为：

  1. 列出题目给出的一些关系式，明确未知数和已知数。
  2. 对关系式进行变化，将关系式转化成向量相乘的形式。
  3. 构造格，用 Hermite 定理进行大小验证，若不满足，考虑重新造格。
  4. 格基约化求解，得到未知数。

一

from Crypto.Util.number import *
import random

flag = b'******'
m = bytes_to_long(flag)

a = getPrime(1024)
b = getPrime(1536)

p = getPrime(512)
q = getPrime(512)
r = random.randint(2**14, 2**15)
assert ((p-r) * a + q) % b < 50

c = pow(m, 65537, p*q)

print(f'c = {c}')
print(f'a = {a}')
print(f'b = {b}')

我们可以从题目里拿到以下关系：

(p−r)⋅a+q+k⋅b=x<50(p-r)\cdot a+q+k\cdot b = x < 50 (p−r)⋅a+q+k⋅b=x<50

c≡m65537(mod(pq))c\equiv m^{65537}\pmod{(pq)} c≡m65537(mod(pq))

a,ba,ba,b是已知的，p,q,rp,q,rp,q,r是未知的。然后后面的加密是正常的 RSA 加密手法。

看看大小：

∣a∣=21024,∣b∣=21536,∣p∣=2512,∣q∣=2512,214<∣r∣<215|a|=2^{1024},|b|=2^{1536},|p|=2^{512},|q|=2^{512},2^{14}<|r|<2^{15} ∣a∣=21024,∣b∣=21536,∣p∣=2512,∣q∣=2512,214<∣r∣<215

稍微移一下项，我们忘格子的形式上靠：

(p−r)⋅a+k⋅b=x−q(p-r)\cdot a+k\cdot b=x-q (p−r)⋅a+k⋅b=x−q

考虑构造以下格子：

L=[1a0b]L=\begin{bmatrix} 1 & a\\ 0 & b \end{bmatrix} L=[10​ab​]

这样可以找到：

(p−r,k)L=(p−r,x−q)=v(p-r,k)L=(p-r,x-q)=\mathbf{v} (p−r,k)L=(p−r,x−q)=v

计算 v\mathbf{v}v 的大小：

∣∣v∣∣=∣p−r∣2+∣x−q∣2≈2⋅2512<2⋅∣b∣1/2||\mathbf{v}||=\sqrt{|p-r|^2+|x-q|^2}\approx\sqrt2\cdot2^{512}<\sqrt2\cdot |b|^{1/2} ∣∣v∣∣=∣p−r∣2+∣x−q∣2​≈2​⋅2512<2​⋅∣b∣1/2

大小可以，开归！然后就是正常的 RSA，进行一个爆破的解密。

        P = p + r
        Q = q + x
        phi = (P - 1)*(Q - 1)
        if gcd(phi, e) != 1:
            continue
        d = inverse_mod(e, phi)
        m = power_mod(c, d, P*Q)
        if b'NSSCTF' in long_to_bytes(int(m)):
            print(long_to_bytes(int(m)))
            flag = True
            break
    if flag:
        break

from Crypto.Util.number import *

flag = b'******' # uuid形式

r = getPrime(175)
a = inverse(r, p)

print(f'a = {a}')
print(f'p = {p}')

from Crypto.Util.number import *
from gmpy2 import *

flag = b'******'
m = bytes_to_long(flag)

assert m.bit_length() == 351
p = getPrime(1024)
b = getPrime(1024)
c = getPrime(400)

a = (b*m + c) % p

print(f'a = {a}')
print(f'b = {b}')
print(f'p = {p}')

拿条件：

a=bm+c+kpa=bm+c+kp a=bm+c+kp

a−bm−kp=ca-bm-kp=c a−bm−kp=c

可以构造格：

L=[10a01−b00p]L=\begin{bmatrix} 1 & 0 & a\\ 0 & 1 & -b\\ 0 & 0 & p \end{bmatrix} L=​100​010​a−bp​​

有：

(1,m,−k)L=(1,m,c)=v(1,m,-k)L=(1,m,c)=\mathbf{v} (1,m,−k)L=(1,m,c)=v

看大小：

∣∣v∣∣=1+∣m∣2+∣c∣2≈2401>∣p∣1/3=2341||\mathbf{v}||=\sqrt{1+|m|^2+|c|^2}\approx 2^{401}>|p|^{1/3}=2^{341} ∣∣v∣∣=1+∣m∣2+∣c∣2​≈2401>∣p∣1/3=2341

归不了一点！这下不得不优化一下凑个数据了：

先看看差多少：401-341=60，算上三次方得补上 60*3=180。这个补充对向量大小影响不大，这里考虑冗余先补个 200 试试看。

L=[22000a01−b00p]L=\begin{bmatrix} 2^{200} & 0 & a\\ 0 & 1 & -b\\ 0 & 0 & p \end{bmatrix} L=​220000​010​a−bp​​

看大小：

∣∣v∣∣=2200×2+∣m∣2+∣c∣2≈2401<∣2200p∣1/3=2408||\mathbf{v}||=\sqrt{2^{200\times2}+|m|^2+|c|^2}\approx 2^{401}<|2^{200}p|^{1/3}=2^{408} ∣∣v∣∣=2200×2+∣m∣2+∣c∣2​≈2401<∣2200p∣1/3=2408

格基规约，启动！
